# Heroes Task Solution

Реализация алгоритмов для пошаговой стратегии Heroes.

## 1. Генерация армии (GeneratePresetImpl)
**Алгоритм:** Жадный алгоритм.
1. Сортируем типы юнитов по "эффективности" (сумма атаки и здоровья, деленная на стоимость).
2. Набираем максимально возможное количество самых эффективных юнитов (не более 11 шт.), затем переходим к следующим.
3. Координаты выбираются случайно из свободных клеток.

**Сложность:** O(T log T), где T — количество типов юнитов.
* Сортировка типов: `O(T log T)`.
* Цикл набора армии: `O(N)`, где N — итоговое количество юнитов (максимум 11 * T).
* Так как T (4 типа) и N (44 юнита) — константы, алгоритм работает мгновенно.

## 2. Симуляция боя (SimulateBattleImpl)
**Алгоритм:** Пошаговая симуляция.
В каждом раунде все живые юниты сортируются по инициативе (Атака) и совершают ход.

**Сложность:** O(N^2 * log N) на раунд.
* Сортировка юнитов: `O(N log N)`, где N — число живых юнитов.
* Проход по списку юнитов: `O(N)`.
* Внутри хода каждого юнита может вызываться поиск пути (BFS), что добавляет сложность, но сама логика симуляции (без учета сложности поиска пути) — `O(N log N)`.

## 3. Поиск целей (SuitableForAttackUnitsFinderImpl)
**Алгоритм:** Поиск крайнего юнита в ряду.
Проверяются 3 ряда. В зависимости от стороны атаки выбирается либо юнит с минимальной координатой X (слева), либо с максимальной (справа).

**Сложность:** O(N), где N — общее число юнитов.
* Мы проходим по всем рядам и всем юнитам в них ровно один раз.
* Сложность линейная относительно количества юнитов.

## 4. Поиск пути (UnitTargetPathFinderImpl)
**Алгоритм:** BFS (Поиск в ширину).
Используется для нахождения кратчайшего пути в невзвешенном графе (сетка поля). Препятствия хранятся в HashSet для быстрого доступа.

**Сложность:** O(W * H), где W — ширина (27), H — высота (21).
* В худшем случае алгоритм посетит каждую клетку поля один раз.
* Поиск в HashSet занимает O(1).
* Итоговая сложность пропорциональна количеству клеток на поле.
